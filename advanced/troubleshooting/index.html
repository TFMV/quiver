    Troubleshooting¶

    Uh oh! Something's not quite right with your vector database? Don't worry, we've got your back! This guide will help you diagnose and fix common issues with Quiver. Let's turn those frowns upside down! üîç

    Common Issues¶

    Installation Problems¶

    Go Installation Issues¶

    Symptom: Error when installing Quiver with Go.

    go: downloading github.com/weaviate/quiver v0.1.0
go get: github.com/weaviate/quiver@v0.1.0: parsing go.mod:
 module declares its path as: github.com/weaviate/quiver
         but was required as: github.com/your-username/quiver

    Solution: Make sure you're using the correct import path. If you've forked the repository, update your go.mod file:

    go mod edit -replace github.com/weaviate/quiver=github.com/your-username/quiver@v0.1.0

    Python Installation Issues¶

    Symptom: Error when installing the Python package.

    ERROR: Could not build wheels for quiver which use PEP 517 and cannot be installed directly

    Solution: Make sure you have the latest pip and build tools:

    pip install --upgrade pip setuptools wheel

    Startup Issues¶

    Storage Path Problems¶

    Symptom: Error when starting Quiver with a storage path.

    failed to open DuckDB: error creating new DuckDB database: Internal: IO Error: Could not read from file... Is a directory

    Solution: Make sure you're providing a file path, not just a directory:

    // Incorrect
idx, err := quiver.New(quiver.Config{
    StoragePath: "/tmp/quiver",
}, logger)

// Correct
idx, err := quiver.New(quiver.Config{
    StoragePath: "/tmp/quiver/quiver.db",
}, logger)

    Permission Issues¶

    Symptom: Permission denied errors when starting Quiver.

    failed to open DuckDB: error creating new DuckDB database: Internal: IO Error: Permission denied

    Solution: Make sure the user running Quiver has write permissions to the storage directory:

    # Give write permissions
chmod 755 /path/to/storage/directory

    Vector Operations¶

    Dimension Mismatch¶

    Symptom: Error when adding vectors with incorrect dimensions.

    vector dimension mismatch: expected 128, got 64

    Solution: Make sure all vectors have the same dimension as specified in the configuration:

    // Make sure these match
idx, err := quiver.New(quiver.Config{
    Dimension: 128,
}, logger)

// Vector must have 128 dimensions
vector := make([]float32, 128)

    Invalid Vector IDs¶

    Symptom: Error when using invalid vector IDs.

    vector ID 0 is reserved and cannot be used

    Solution: Vector IDs must be positive integers and cannot be 0:

    // Incorrect
idx.Add(0, vector, metadata)

// Correct
idx.Add(1, vector, metadata)

    Search Issues¶

    No Results Returned¶

    Symptom: Search returns no results even though vectors are added.

    Solution: Check the following:

    1. Make sure you've added vectors to the index
    2. Check if your search parameters are too restrictive
    3. Verify that your query vector is normalized if your index uses cosine similarity
    4. If using filters, make sure they're not too restrictive
    // Try increasing k
results, err := idx.Search(queryVector, 100, 1, 10)

// Try increasing efSearch
idx, err := quiver.New(quiver.Config{
    HNSWEfSearch: 200, // Default is 100
}, logger)

    Poor Search Quality¶

    Symptom: Search results don't match expectations or have low similarity scores.

    Solution: Tune your HNSW parameters:

    idx, err := quiver.New(quiver.Config{
    HNSWM:           32,    // Default is 16, higher means more connections
    HNSWEfConstruction: 200, // Default is 100, higher means better index quality
    HNSWEfSearch:    100,   // Default is 100, higher means better search quality
}, logger)

    Metadata Issues¶

    SQL Syntax Errors¶

    Symptom: Error when using filters with incorrect SQL syntax.

    failed to execute query: syntax error at or near "WHERE"

    Solution: Make sure your SQL filter is valid:

    // Incorrect
results, err := idx.SearchWithFilter(queryVector, 10, "WHERE name = 'test'")

// Correct
results, err := idx.SearchWithFilter(queryVector, 10, "name = 'test'")

    Type Conversion Errors¶

    Symptom: Error when retrieving metadata with incorrect types.

    cannot convert metadata field 'age' from string to int

    Solution: Make sure you're using consistent types for metadata fields:

    // When adding
metadata := map[string]interface{}{
    "age": 30, // Use int consistently
}
idx.Add(1, vector, metadata)

// When retrieving
age := result.Metadata["age"].(int) // Cast to int

    Performance Issues¶

    Slow Addition¶

    Symptom: Adding vectors is slower than expected.

    Solution: Use batch operations and tune batch size:

    // Use batch operations
vectors := make([][]float32, 1000)
ids := make([]uint64, 1000)
metadata := make([]map[string]interface{}, 1000)
// Fill vectors, ids, and metadata...
idx.BatchAdd(ids, vectors, metadata)

// Tune batch size
idx, err := quiver.New(quiver.Config{
    BatchSize: 5000, // Default is 1000
}, logger)

    Slow Search¶

    Symptom: Search operations are slower than expected.

    Solution: Tune search parameters and consider using in-memory storage:

    // Use in-memory storage for faster search
idx, err := quiver.New(quiver.Config{
    StoragePath: ":memory:",
}, logger)

// Reduce efSearch for faster (but potentially less accurate) search
idx, err := quiver.New(quiver.Config{
    HNSWEfSearch: 50, // Default is 100
}, logger)

    High Memory Usage¶

    Symptom: Quiver is using more memory than expected.

    Solution: Tune memory usage parameters:

    idx, err := quiver.New(quiver.Config{
    // Reduce batch size to lower memory usage during additions
    BatchSize: 500, // Default is 1000

    // Use disk storage instead of memory
    StoragePath: "/path/to/storage.db",
}, logger)

    Debugging Techniques¶

    Enabling Debug Logging¶

    To get more detailed information about what's happening inside Quiver, enable debug logging:

    // Create a development logger with debug level
config := zap.NewDevelopmentConfig()
config.Level = zap.NewAtomicLevelAt(zap.DebugLevel)
logger, _ := config.Build()

// Pass the logger to Quiver
idx, err := quiver.New(quiver.Config{
    // Your config here
}, logger)

    Inspecting the Index¶

    You can inspect the index to verify its state:

    // Get index stats
stats, err := idx.Stats()
fmt.Printf("Index stats: %+v\n", stats)

// Check if a vector exists
exists, err := idx.Exists(vectorID)
fmt.Printf("Vector %d exists: %v\n", vectorID, exists)

// Get a specific vector
vector, metadata, err := idx.Get(vectorID)
fmt.Printf("Vector %d: %v, Metadata: %v\n", vectorID, vector, metadata)

    Validating Metadata¶

    To validate that your metadata is stored correctly:

    // Add a vector with metadata
metadata := map[string]interface{}{
    "name": "test",
    "age": 30,
}
idx.Add(1, vector, metadata)

// Retrieve and validate
vector, retrievedMetadata, err := idx.Get(1)
fmt.Printf("Original metadata: %v\n", metadata)
fmt.Printf("Retrieved metadata: %v\n", retrievedMetadata)

// Check specific fields
if retrievedMetadata["name"] != metadata["name"] {
    fmt.Println("Metadata name mismatch!")
}

    Benchmarking¶

    If you're experiencing performance issues, run benchmarks to identify bottlenecks:

    // Simple benchmark function
func benchmarkSearch(idx *quiver.Index, queryVector []float32, k int) time.Duration {
    start := time.Now()
    idx.Search(queryVector, k, 1, 10)
    return time.Since(start)
}

// Run benchmark
durations := make([]time.Duration, 100)
for i := 0; i < 100; i++ {
    durations[i] = benchmarkSearch(idx, queryVector, 10)
}

// Calculate average
var total time.Duration
for _, d := range durations {
    total += d
}
average := total / time.Duration(len(durations))
fmt.Printf("Average search time: %v\n", average)

    For more detailed benchmarking, see the Benchmarking Guide.

    Troubleshooting HTTP API¶

    Connection Issues¶

    Symptom: Cannot connect to the HTTP API.

    Solution: Check the following:

    1. Verify the server is running: ps aux | grep quiver
    2. Check the listening port: netstat -tuln | grep 8080
    3. Ensure there are no firewall issues: curl -v http://localhost:8080/health

    Authentication Issues¶

    Symptom: Receiving 401 Unauthorized errors.

    Solution: Make sure you're providing the correct API key:

    # Incorrect
curl -X POST http://localhost:8080/vectors

# Correct
curl -X POST http://localhost:8080/vectors -H "Authorization: Bearer your-api-key"

    Request Format Issues¶

    Symptom: Receiving 400 Bad Request errors.

    Solution: Validate your request format:

    # Use jq to validate JSON
echo '{"id": 1, "vector": [0.1, 0.2, 0.3], "metadata": {"name": "test"}}' | jq

# Check your request with verbose output
curl -v -X POST http://localhost:8080/vectors \
  -H "Content-Type: application/json" \
  -d '{"id": 1, "vector": [0.1, 0.2, 0.3], "metadata": {"name": "test"}}'

    Troubleshooting Python Client¶

    Import Errors¶

    Symptom: Error when importing the Python client.

    ImportError: No module named quiver

    Solution: Make sure the package is installed and in your Python path:

    pip install quiver-db
python -c "import quiver; print(quiver.__version__)"

    Connection Issues¶

    Symptom: Cannot connect to Quiver from Python.

    ConnectionError: Failed to connect to Quiver server

    Solution: Check the server URL and authentication:

    # Make sure the URL is correct
client = quiver.Client("http://localhost:8080", api_key="your-api-key")

# Test the connection
try:
    client.health()
    print("Connection successful!")
except Exception as e:
    print(f"Connection failed: {e}")

    Advanced Troubleshooting¶

    Database Corruption¶

    Symptom: Errors indicating database corruption.

    failed to open DuckDB: error opening existing DuckDB database: Internal: Database corrupted

    Solution: Try to recover the database or create a new one:

    # Backup the corrupted database
cp /path/to/quiver.db /path/to/quiver.db.bak

# Create a new database
rm /path/to/quiver.db

    Memory Leaks¶

    Symptom: Memory usage grows over time without releasing.

    Solution: Use Go's built-in profiling tools:

    import _ "net/http/pprof"

func main() {
    // Start pprof server
    go func() {
        http.ListenAndServe("localhost:6060", nil)
    }()

    // Your Quiver code here
}

    Then analyze with:

    go tool pprof http://localhost:6060/debug/pprof/heap

    Deadlocks¶

    Symptom: Operations hang indefinitely.

    Solution: Use Go's deadlock detector:

    # Set deadlock detection environment variable
GODEBUG=deadlock=1 ./your-quiver-app

    Getting Help¶

    If you're still stuck after trying these troubleshooting steps, don't hesitate to reach out:

    1. GitHub Issues: Open an issue on the Quiver GitHub repository
    2. Discord: Join our community on Discord
    3. Stack Overflow: Ask a question with the quiver tag

    When asking for help, please include:

    • Quiver version
    • Go/Python version
    • Operating system
    • Minimal code example that reproduces the issue
    • Error messages and logs
    • Any relevant configuration
